4-DOF Arduino Robotic Arm – Iteration 1

Mechanical Design | Embedded Systems | Control Logic

Objective

The goal of this project was to design and build a low-cost 4-degree-of-freedom robotic arm to explore mechanical design, embedded systems, and control logic. This first iteration focuses on learning servo control, basic sensor integration, and modular software design.

System Overview

Controller: Arduino Nano
Actuators: 4 micro servos (Base, Shoulder, Elbow, Gripper)
Input: IR Remote
Sensor: Ultrasonic distance sensor
Power: 5V, 2A supply

Block Diagram:

IR Remote → Arduino Nano → Servo Motors → Robotic Arm Joints
Ultrasonic Sensor → Arduino → Gripper Logic

Mechanical Design

  • 3D printed arm links (PLA)
  • Servo Horns:
    • Cross horn → Base rotation (high torque)
    • 2 Single-arm horns → Shoulder and Elbow
    • Modified double-arm horn → Gripper (cut into rectangular tab for gear interface)
  • Gripper tension: Thick rubber band
  • Gripper attached via M3 screw + nut

Key Design Choices: Modified horn created a keyed mechanical interface for torque transfer. Hot glue was used for quick assembly and rework.

Electronics

  • Arduino Nano controls all servos and receives IR commands
  • IR Receiver decodes commands from remote
  • Ultrasonic Sensor triggers automated gripper motion
  • Servo power managed with separate 5V 2A supply

Software Architecture

  • Modular code: servo motion, gripper logic, ultrasonic measurement separated into functions
  • Servo angle tracking stored in variables for smooth incremental movement
  • IR remote mapped to directional movement and gripper control
  • Auto-gripper mode triggered by ultrasonic sensor

Challenges & Lessons Learned

  • Continuous rotation servos cannot hold precise angles → positional servos required
  • Hot glue bond limited torque capacity
  • Timing delays needed to match physical servo response
  • IR input decoding requires careful mapping and testing

Future Improvements

  • Replace glue with mechanical fastening for higher torque
  • Implement inverse kinematics for precise control
  • Add joystick or serial input control
  • Smooth servo motion using non-blocking code
